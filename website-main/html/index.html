Project Overview

In hospitals and pharmaceutical labs, controlled substances like opioids and research materials require secure access to prevent misuse, theft, or tampering.

PharmaGate is a smart double-door security system that introduces multi-layer authentication and environmental awareness for high-security zones. The system combines QR-based app authentication, biometric verification (face and fingerprint), tailgating prevention, and fire-response mechanisms. It also includes real-time data monitoring and visualization using ThingSpeak, ensuring security logs are transparent and auditable.

By enhancing physical and digital access control, PharmaGate improves both operational safety and regulatory compliance in sensitive medical environments.

How It Works

PharmaGate is a smart double-door access control system designed for high-security environments like hospitals and pharmaceutical labs. The system starts with QR code verification using a mobile app, followed by biometric authentication such as face ID and fingerprint scanning. Environmental sensors like IR, PIR, and ultrasonic are used to detect presence and prevent tailgating. In the event of a fire, the system triggers emergency unlocks based on real-time temperature and humidity readings. All access logs and alerts are pushed to the ThingSpeak cloud platform for secure visualization and traceability.

Key Features

Ensures sequential access using two separate validation stages.

App-Based QR Login

Access codes are validated securely through a mobile application.

Biometric Verification

Face ID and fingerprint verification before unlocking second door.

Tailgating Detection

Ultrasonic and IR sensors detect if multiple people enter improperly.

Fire Detection Auto-Unlock

Temperature and humidity sensors trigger emergency evacuation mode.

Cloud Dashboard (ThingSpeak)

Real-time access logs and alerts visualized securely on the cloud.

Challenges & Solutions

Challenge 1: Real-Time QR Verification

Ensuring QR codes generated on the app are immediately recognized and verified by the door system.

Solution:

Integrated Firebase Realtime Database with asynchronous checking on ESP8266 using polling and timed validation.

Challenge 2: Sensor Fusion Accuracy

Different sensors gave inconsistent readings in crowded environments.

Solution:

Combined multiple sensor types (IR + Ultrasonic + PIR) to confirm occupancy and reduce false positives.

Challenge 3: Biometric Module Integration

Difficulty in matching fingerprint/face data with existing authentication flow.

Solution:

Developed a separate verification module with a pass-through validation signal to unify the access logic.

Challenge 4: Data Visualization & Logging

Storing and viewing user access logs securely in real-time.

Solution:

Used ThingSpeak with public/private API keys and time-series widgets to log, protect, and view events.

Possible Methods to Compromise Our Systems

This section will be explaining what are the possible attacks that we will be facing.

1. Network Scanning

Discover devices on the same network as the target. It helps identify the ESP8266 (Door A) and any other devices connected to the network.

Tools: Nmap (Command-line network scanner: nmap -sn XXX.XXX.XXX.XXX/XXX)
Requirements:
• Access to the Same Local Network
• No Network Segmentation
• Device Active on Network

2. Wi-Fi Sniffing

Captures Wi-Fi traffic to identify the ESP8266's IP address and other details, provided the network uses weak encryption or no encryption at all.

Tools: Wireshark (Network protocol analyzer)
Requirements:
• Open or Weakly Encrypted Wi-Fi Network
• Wi-Fi Sniffing Tools
• Access to Network

3. Social Engineering

This method involves manipulating people within the organization to gain information or physical access to the system.

Tools: Phishing (Fake login pages or emails)
Requirements:
• Physical Access
• Tricking Personnel

4. Brute Forcing Firebase Authentication

This attack involves attempting to guess or brute-force the credentials for accessing Firebase (or any associated app).

Tools: Hydra (Brute-force password cracking tool)
Requirements:
• Weak Authentication Mechanisms
• Access to Firebase Database
• List of Common Credentials

5. Exploit Weaknesses in HTTP/HTTPS Communication

A Man-in-the-Middle (MITM) attack that intercepts unencrypted HTTP traffic or weakly encrypted HTTPS traffic between the ESP8266 and the Android app.

Tools: MITMproxy (HTTP/HTTPS interception tool)
Requirements:
• Unencrypted HTTP or Weak HTTPS
• Proxy Setup
• Device on the Same Network

6. P2P Communication Exploitation

This attack targets unencrypted or weakly encrypted P2P communication between devices, allowing the attacker to intercept or spoof the communication.

Tools: Wireshark (Packet sniffing tool)
Requirements:
• Unencrypted P2P Communication
• Network Access

7. Physical Attack (Emergency Exit Override)

This involves using physical access to trigger the emergency exit (e.g., bypassing RFID or pressing a button) or tampering with the hardware.

Tools: RFID Reader/Writer (To clone RFID tags)
Requirements:
• Physical Access
• Valid RFID or Button Signal

Future Scope

The current implementation of PharmaGate establishes a solid foundation for secure access control in critical environments. Below are some areas we envision expanding into for future iterations.

  • Expand biometric options with voice recognition or retina scan.
  • Integrate mobile notifications and alert escalation protocols.
  • Implement access analytics and behavior pattern recognition using AI.
  • Enable dynamic access privileges based on time or location.
  • Support large-scale hospital integration with centralized dashboards.
