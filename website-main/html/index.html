PharmaGate

Securing Controlled Drug Access in Hospitals and Labs

üìå Project Overview

In hospitals and pharmaceutical labs, controlled substances like opioids and research materials require secure access to prevent misuse, theft, or tampering.

PharmaGate is a smart double-door security system that introduces multi-layer authentication and environmental awareness for high-security zones. The system combines QR-based app authentication, biometric verification (face and fingerprint), tailgating prevention, and fire-response mechanisms. It also includes real-time data monitoring and visualization using ThingSpeak, ensuring security logs are transparent and auditable.

By enhancing physical and digital access control, PharmaGate improves both operational safety and regulatory compliance in sensitive medical environments.

‚öôÔ∏è How It Works

PharmaGate is a smart double-door access control system designed for high-security environments like hospitals and pharmaceutical labs. The system starts with QR code verification using a mobile app, followed by biometric authentication such as face ID and fingerprint scanning. Environmental sensors like IR, PIR, and ultrasonic are used to detect presence and prevent tailgating. In the event of a fire, the system triggers emergency unlocks based on real-time temperature and humidity readings. All access logs and alerts are pushed to the ThingSpeak cloud platform for secure visualization and traceability.

üõ†Ô∏è Challenges & Solutions

During the development of PharmaGate, we encountered several technical and design challenges. Here are a few key ones and how we addressed them.

üöß Challenge 1: Real-Time QR Verification

Ensuring QR codes generated on the app are immediately recognized and verified by the door system.

‚úÖ Solution:

Integrated Firebase Realtime Database with asynchronous checking on ESP8266 using polling and timed validation.

üöß Challenge 2: Sensor Fusion Accuracy

Different sensors gave inconsistent readings in crowded environments.

‚úÖ Solution:

Combined multiple sensor types (IR + Ultrasonic + PIR) to confirm occupancy and reduce false positives.

üöß Challenge 3: Biometric Module Integration

Difficulty in matching fingerprint/face data with existing authentication flow.

‚úÖ Solution:

Developed a separate verification module with a pass-through validation signal to unify the access logic.

üöß Challenge 4: Data Visualization & Logging

Storing and viewing user access logs securely in real-time.

‚úÖ Solution:

Used ThingSpeak with public/private API keys and time-series widgets to log, protect, and view events.

üîì Step-by-Step: Unlocking Door A

Step 1: LCD Displays Access Code

The LCD screen mounted on Door A will display a randomly generated 6-digit code.

Step 2: Open the App

The user opens our custom mobile app and logs in using their registered account.

Step 3: Enter Code

Using the app, the user enters the 6-digit code shown on the door's LCD screen.

Step 4: Verification

The app sends the code to a Firebase database. The ESP8266 on Door A reads and verifies it in real time.

Step 5: Door A Unlocks

If the code and user match, the green LED lights up and Door A unlocks.
